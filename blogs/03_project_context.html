Project Context

The project described here will be implemented within the clinical modelling methodology.

Generally, this methodology consists of several steps. It begins with the identification of a need for a clinical model, based on a real clinical necessity. A multidisciplinary team conducts an analysis to determine all the necessary data elements. Consensus should be achieved by the team, drawing on professional experience and existing guidelines (such as the Minimum Data Set for Clinical Reports, for example).

Once consensus is reached, the data structure is defined in the design phase. Here, the structure is specified, determining the occurrences and location of every element. This is guided not only by clinical necessity but also by technological frameworks. During the development phase, this structure is implemented into archetypes and templates using openEHR standards.

When the templates are ready for publication, a validation process is conducted to ensure correct function from both clinical and technical perspectives. If validation is successful, the archetypes and templates are published in the international Clinical Knowledge Manager (CKM).


Based on Moner D. et al.

Although real-life development is not always as linear as the schema suggests, there is a critical moment: when the data elements for the data structure are defined, professionals must search for archetypes that might contain these data elements. This is important, as the philosophy of openEHR is based on the reuse of assets and worldwide consensus on reusable archetypes.

At this stage, the professional must determine:

  • Which archetypes already exist that fit the data elements?
  • Which existing archetypes need to be adapted (change request)?
  • Which archetypes need to be created anew?

The proposed project aims to assist professionals in this critical step, given the complexity of the archetype ecosystem.
