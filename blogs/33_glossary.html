The openEHR Class Glossary

Problem Statement

The openEHR specifications are complex and distributed across multiple webpages, making them difficult to navigate. For example, discovering all attributes of a class, including inherited ones, often requires switching between pages, which can quickly become confusing. Additionally, research into classes and their attributes is typically unidirectional. If a user wants to perform a reverse search—such as finding all classes that contain an element of class PARTICIPATION—this is essentially impossible with the current resources.

Existing Solutions

Tools like NeoEHR UML improve the visibility and navigability of the specifications by providing UML diagrams. However, these tools still have limitations, particularly regarding advanced search capabilities.

A text version of the reference model is available in the openEHR GitHub repository: openEHR/specifications-ITS-BMM but it does not contain all the information required for comprehensive search and navigation.

Proposed Solution

To address these issues, we generate a comprehensive reference model and store all relevant information in a JSON file. This format is easy to parse and enables a variety of search operations, including reverse lookups.

Implementation

  1. Data Extraction:
    • A Python script connects to the following openEHR specification webpages:
      • BASE Types
      • Common
      • Data Structures
      • Data Types
      • Demographic
      • EHR Extract
      • EHR
      • Integration
      • Support
    • The script extracts all tables from these pages. If a table represents a class, all relevant information is parsed and stored.
    • The extraction process is automated to update the glossary dataset on a monthly basis.
  2. Glossary Search Interface:
    • A dedicated HTML page was added to the Archetype Companion tool, enabling users to search the reference model data.
    • Search capabilities include class name, attribute name, attribute description, and attribute data type.

Features

  • Comprehensive, up-to-date JSON dataset of the openEHR reference model
  • Automated monthly updates via Python scripting
  • Advanced search functionality, including reverse lookups
  • Seamless integration with the Archetype Companion for easy access
