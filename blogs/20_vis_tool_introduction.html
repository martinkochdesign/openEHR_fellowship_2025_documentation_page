Visualization Tool - Introduction

Once the archetype information has been extracted in the dataset.js file, the second part of the project begins.

The general idea is to design a tool that enhances the way to search, plan, and visualize openEHR archetypes, providing a comprehensive platform for clinical knowledge modellers to efficiently manage archetype data.

General Description

The Archetype Visualization and Planning Tool aims to provide detailed information on existing archetypes and facilitate the addition of future archetypes to the ecosystem.

The main development goals are:

  • Provide comprehensive information on existing archetypes from an archetype source.
  • Facilitate the fast search of existing archetypes.
  • Help to understand the archetype ecosystem by network visualization.
  • Enable the planning of addition of new archetypes to the ecosystem.

The key features can be broken down into:

  • Archetype Search
  • Archetype Project Management
  • Network Visualization

The following sections provide detailed descriptions of the key features as currently envisioned. The proposal is made with the best intentions but should be taken "as is." Some points may change due to new opportunities or technical limitations found during development.

Key Features

Archetype Search

The archetype search should be a simple search algorithm that takes the data from the dataset and filters out all of the archetypes that do not contain the search text.

The following features for the search should be considered:

  • Search input over a text field.
  • Search result update at every keystroke in the text field (Does not need a separate "Search" button).
  • Search should be fast.
  • Search archetypes. The search should be done by filtering one or more of the following:
    • keywords
    • class
    • purpose
    • concept name
    • archetype ID
  • Search is only executed over the existing archetypes from the "dataset.js" file.

Additional functionalities that might be considered are:

  • Adding regex or regex-like expressions to the search algorithm, so wildcards ("*") can be used.
  • Sort search results alphabetically (A-Z, Z-A).
  • Show number of found results.
  • Show preview of the text, where the search text was found.

Archetype Project Management

The project management component of the tool should enable users to manage collections of archetypes and track development progress.

The following features for the project management should be considered:

  • Archetype Management
    • Existing Archetypes: Archetypes from the dataset.js file.
      • Display information on archetype after clicking on it. The information of an archetype should include:
        • Archetype ID
        • Concept name
        • Purpose
        • Items
        • Inclusions
        • Exclusions
        • Parent
        • Children
        • Description
        • State
        • Date
        • Keywords
        • Recommended archetypes
      • Alerts for archetype predrafts or drafts usage.
      • Suggestion of similar and relevant archetypes.
    • New Archetypes: Planned archetypes added by the user.
      • Only outlines of archetypes, not fully functional ADL1.4 archetypes.
      • Are added to the collection. They only exist in the collection context.
      • Display editable information on archetype after clicking on it:
        • Archetype name
        • Class
        • Purpose
        • Keywords
        • Assigned elements
        • Included archetypes. Only archetypes from the collection can be included.
    • Collection of Archetypes
      • New and existing archetypes should be bundled into a collection by the user.
      • Existing archetypes can be added or removed from the collection.
      • New archetypes are created in the collection and get deleted when removed from the collection.
  • Checklist Management
    • Structured Checklist
      • Structured in sections and elements. One level deep (no section within a section).
      • Sections and elements are movable.
      • Sections and elements can be added/deleted.
      • Assigned archetype per element.
      • "Completed" marker per element.
      • "Change request" marker per element.
    • Load/save capabilities in format XLSX.
  • Project Management
    • Save/load capabilities in JSON format.
    • Saves project metadata, checklist data, new archetypes, and archetype collection.
    • Delete project data.

Additional functionalities that might be considered are:

  • Project metadata might include:
    • Author name
    • Author affiliation
    • Copyright
    • Description
    • Date of generation
  • New Archetype
    • Auto-suggest from keyword pool when typing in keywords.

Network Visualization

The following features for the network visualization should be considered:

  • A network visualization using nodes (circles) and edges (connecting lines).
  • Clicking on a node should select it and display its information for convenience.
  • Basic navigation around the network should be implemented: Move and zoom.
  • Visualization Modes:
    • Network visualization of archetypes and their connections.
      • Visualize parent-child relationship (specialized archetypes).
      • Visualize inclusion/exclusion relationships.
    • Network visualization of archetype collection.
    • Visualize new and existing archetypes.
    • Visualize inclusion if exists between members of collection.
    • Visualize specialization if exists between members of collection.
    • Visualization of recommended or similar archetypes.
    • Visualization of project/checklist.
      • Show sections.
      • Show elements of sections and connect to corresponding section.
      • Show archetypes from collection, if they have been assigned to an element. Show the connection between archetype and element.

Other Requirements

Apart from the key functionalities, a few additional requirements can be already determined.

  • About Page
    • The About page will provide an overview of the tool, its purpose, and the team behind its development.
  • Accessibility:
    • Cross-platform use (via web browser): Ensuring cross-platform compatibility will allow users to access the tool from various devices and browsers, enhancing usability and reach.
    • No user account management: By not requiring user account management, the tool simplifies access and reduces potential security concerns related to user data storage.
  • User Interface and Experience:
    • Intuitive navigation and responsive design to enhance user experience.
    • Incorporate user feedback mechanisms for continuous improvement.
  • Performance:
    • Optimize load times and responsiveness, especially when handling large datasets.
  • Integration:
    • Support integration with other systems or tools as needed.
  • Documentation and Support:
    • Provide comprehensive documentation and support resources to assist users.

These requirements might change over time, but special care is going to be given to fulfill these development goals.
